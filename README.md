# 小白验证Vue3卡密登录系统 - Vue Uniapp 项目

## 项目简介
小白验证xbzhan.com
这是一个基于 Vue 3 + Vite 构建的卡密登录系统，实现了以下功能：

- 卡密登录验证
- 心跳连接状态监控
- 单独的功能区页面
- 未登录访问控制
- 响应式设计，支持移动端

## 项目结构

```
uniapp-cardkey-login/
├── dist/              # 构建输出目录
├── public/            # 静态资源目录
├── src/
│   ├── assets/        # 资源文件
│   ├── router/        # 路由配置
│   ├── views/         # 页面组件
│   │   ├── LoginView.vue      # 登录页面
│   │   └── FunctionsView.vue  # 功能区页面
│   ├── api.js         # API管理类
│   ├── App.vue        # 应用入口组件
│   ├── main.js        # 应用入口文件
│   └── style.css      # 全局样式
├── api.php            # API代理文件
├── index.html         # HTML模板
├── package.json       # 项目配置
└── vite.config.js     # Vite配置
```

## 核心功能

### 1. 卡密登录验证
- 支持卡密输入和验证
- 连接服务器进行卡密验证
- 显示验证结果和错误信息

### 2. 心跳连接状态
- 登录后每60秒自动发送心跳请求
- 实时显示连接状态（已连接/连接中断）
- 确保与服务器的持续连接

### 3. 功能区页面
- 独立的功能区页面，显示用户信息
- 功能列表展示
- 退出登录功能

### 4. 未登录访问控制
- 路由守卫，未登录用户无法访问功能区
- 未登录用户访问功能区时会被重定向到登录页
- 功能区页面添加登录状态检查

## 技术栈

- **前端框架**：Vue 3 (Composition API)
- **构建工具**：Vite
- **路由管理**：Vue Router 4
- **样式**：CSS3
- **API请求**：Fetch API
- **状态管理**：localStorage

## 安装和运行

### 安装依赖

```bash
cd uniapp-cardkey-login
npm install
```

### 开发模式运行

```bash
npm run dev
```

### 生产模式构建

```bash
npm run build
```

构建完成后，将 `dist` 目录下的文件部署到服务器即可。

## API 代理

项目使用 `api.php` 文件作为API代理，用于转发请求到实际的API服务器。确保在部署时将 `api.php` 文件复制到与 `index.html` 同级目录。

## 安全说明

- 卡密验证通过加密传输，确保安全性
- 心跳机制确保连接状态正常
- 未登录用户无法访问功能区，保护用户数据

## 响应式设计

项目采用响应式设计，支持在不同设备上的显示：
- 桌面端：功能区采用网格布局，展示更多功能
- 移动端：功能区采用单列布局，优化触摸操作

## 注意事项

1. 确保 `api.php` 文件存在于部署目录中
2. 卡密验证需要连接到实际的API服务器
3. 心跳机制依赖于服务器的响应
4. 本地开发时，可能需要配置跨域访问

## 故障排查

### 1. 登录失败
- 检查卡密是否正确
- 检查网络连接是否正常
- 检查API服务器是否可用

### 2. 连接状态显示为"连接中断"
- 检查网络连接
- 检查API服务器是否正常响应
- 检查心跳请求是否被拦截

### 3. 功能区无法访问
- 确保已登录
- 检查localStorage中的登录状态
- 检查路由守卫配置
